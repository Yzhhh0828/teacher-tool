# 项目全面检查与修复计划

## 已发现的问题

### 高优先级问题
1. **数据库异步提交问题** - `create_class` 中存在两次 commit，应该合并为一次事务
2. **Grade 模型类型标注** - `created_at` 字段缺少 `DateTime` 类型标注
3. **MCP Tools 空学生列表逻辑** - `random_shuffle_seats` 在学生列表为空时会产生除零错误

### 中优先级问题
4. **重复代码** - `check_class_permission` 函数在 4 个文件中重复定义
5. **Agent API 响应格式不一致** - `process` 返回的 dict 包含 `type` 字段，但 API 端期望 `content` 字段
6. **SSE 事件格式问题** - agent chat 端点的 SSE 响应格式可能不正确

### 低优先级问题
7. **缺少测试覆盖** - 部分 API 端点缺少测试覆盖

## 修复计划

### 第一步：修复数据库异步提交问题
- 修改 `class_.py` 中的 `create_class` 函数，将两次 commit 合并为一次

### 第二步：修复 Grade 模型类型标注
- 在 `exam.py` 中为 `Grade.created_at` 添加 `DateTime` 类型标注

### 第三步：修复 MCP Tools 空学生列表逻辑
- 修改 `tools.py` 中的 `random_shuffle_seats` 处理空学生列表的情况

### 第四步：重构重复的 check_class_permission 函数
- 将重复的 `check_class_permission` 函数移到 `deps.py` 中
- 更新所有 API 文件使用统一的函数

### 第五步：修复 Agent API 响应格式
- 确保 agent chain 的响应格式与 API 端点期望一致

### 第六步：最终验证
- 运行所有测试确保修复成功
- 检查代码质量
